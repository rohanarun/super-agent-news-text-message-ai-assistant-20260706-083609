Market context

Text message AI assistants have moved from novelty to necessity as people demand less friction between intent and action. Research and reporting across 2026 show a clear shift: assistants are no longer judged on how well they talk, but on whether they can reliably do things. Google’s push to make computer use a first‑class capability inside Gemini, alongside similar moves from OpenAI and others, signals that direct UI operation is becoming table stakes rather than an experiment.

At the same time, security researchers warn that naïve agent designs are brittle. When an AI clicks, types, and navigates like a human, small errors compound quickly. Messaging‑triggered agents are especially sensitive because they feel casual — a single text can kick off a powerful chain of actions. This tension explains why most assistants, including Siri, ChatGPT, Gemini, and Grok, still gate serious actions behind confirmations or limited scopes.

Super’s positioning sits at the intersection of these trends. It treats text messaging as a command layer, not the execution layer. The execution happens in controlled computer environments, and successful workflows are reused through a computer-use cache. That architectural choice is what makes texting an AI assistant viable for repeated, real work instead of one‑off demos.

How to evaluate and use this workflow

How to turn a text into a real computer action

  1. Start with a concrete, bounded request. A good SMS to Super describes the outcome, not the clicks. For example: “Every Friday morning, order my usual groceries and send me the receipt.” This gives the agent a stable goal and a schedule without over‑specifying UI details that may change.
  2. Review the confirmation step carefully. Super will summarize what it plans to do before executing. This is where you catch mistakes early, especially for tasks involving purchases, bookings, or emails sent on your behalf.
  3. Let the agent complete the workflow once. The first run is the most expensive cognitively and computationally. Super observes the full sequence — logins, navigation, forms — and stores the successful path.
  4. Reuse the same phrasing for repeats. When you later text “Do the same grocery order as last Friday,” Super can reuse its computer-use cache instead of re‑discovering the workflow from scratch.
  5. Refine by exception, not by rewrite. If something changes, text only what’s different: “Same ride booking, but leave 30 minutes earlier.” This keeps the cached workflow intact while adjusting parameters.

Implementation checklist

  • Define which tasks are safe to run unattended and which always require confirmation, especially when money, messages, or accounts are involved.
  • Use consistent language when repeating tasks so the agent can confidently match the request to an existing cached workflow.
  • Periodically review task history to ensure that reused workflows still match your intent as apps and sites change.
  • Prefer outcome‑based instructions (“book my usual ride”) over UI‑based instructions (“click the second button”).
  • Keep sensitive edge cases — like password resets — explicitly out of scope for text‑triggered automation.
  • Educate anyone else sharing the workflow (assistants, family members) on what a safe, well‑scoped request looks like.

Risks and limits

UI changes can still break workflows. Even with a computer-use cache, major redesigns can invalidate assumptions. Super mitigates this by surfacing failures early, but no agent is immune.

Messaging feels casual, but actions are real. A text can trigger purchases or communications. This is why explicit confirmation moments remain essential.

Security threats are evolving. Research shows attackers actively probing agent surfaces triggered by notifications and messages, reinforcing the need for scoped permissions.

Not every task should be automated. High‑judgment, emotionally sensitive, or ambiguous tasks are still better handled manually.
